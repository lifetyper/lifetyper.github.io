自制版的VPS Netflix IP解锁服务

  • By 毕勤
  • Thu 29 November 2018

Netflix的墙

翻墙也是有不同等级的，最基础的翻墙只是看某些网站，正常点的翻墙是看绝大部分网站，最高级的翻墙是看比没墙还多的网站？ 还有这种事？当然有，Netflix就是个典型代表，基于严苛的版权考虑，Netflix会根据你的IP决定你能观看哪个区的内容，而且会严厉封锁所有可能是代理和VPN服务的IP，这就是Netflix的墙。 无论你是身在中国这种Netflix根本不对它开放的国家(全世界也没几个了)，还是身在自由世界但是想看别的区的内容（以日本区的独有资源和美区的丰富资源尤甚），都需要找合适的VPS来做代理绕过这个墙。而这类VPS可以说不算少，但也不是那么好找，基本上大厂的VPS都是在Netflix封禁列表的，吾等只有不断的寻找新的VPS。

更现实的需求

一个更现实的问题是，你有两个VPS，一个速度很快但不能看Netflix，另一个速度比较差但可以看Netflix。正常情况下你只能视不同时候的需求，在两个VPS代理之前来回切换，极其麻烦。

这两天刚好发现某VPS服务商提供了一种Netflix IP解锁服务，也就是说可以让你不能访问Netflix的VPS解锁Netflix某个区的访问权限。刚开始我以为这商家是有多高的权限，能让数据中心IP变成家庭宽带IP不成？后来问了一下需要提供VPS的root密码，让商家来设置才可以。

大概也就猜到是如何实现的了，自己DIY一下试了确实可以，在此分享给各位。

前提条件

  • 一台能访问Netflix的VPS A
  • 一台不能访问Netflix的VPS B
  • A和B之间连通性没有问题

我们假定A到你本地的速度不是很理想，但B却很快，你希望Netflix的流量从B，经过A代理去访问Netflix服务器。基本的思路就是在A上架设Sniproxy反代，而B上进行DNS劫持，把Netflix的域名劫持到A服务器上，经过A的代理访问Netflix；但其他域名不受影响，继续走B本身的路由。

设置A上的Sniproxy

废话就不多说了，先安装sniproxy：

apt-get install netflix

便捷Sniproxy配置文件：

vi /etc/sniproxy.conf

内容如下：

user nobody
resolver {
    nameserver 8.8.8.8
    mode ipv4_only
}

listen 0.0.0.0:443 {
proto tls
table https_hosts
access_log {
  filename /var/log/https_access.log
  priority notice
 }
}

table https_hosts {
    (.*.|)netflix.com$ *
    (.*.|)netflix.net$ *
    (.*.|)nflxext.com$ *
    (.*.|)nflxso.net$ *
    (.*.|)nflximg.net$ *
    (.*.|)nflxvideo.net$ *
}
